# GMP 实训仿真项目演示顺序

> 演示目标：让观看者快速看到项目已经形成“学生学习、教师管理、管理员配置”的闭环，并重点展示前测、个性化学习、实训仿真、教师查看学生数据这几条主线。

## 一、演示前准备

1. 启动项目服务，确认浏览器能访问系统首页。
2. 准备三个角色账号：
   - 学生账号：`123@qq.com`
   - 教师账号：`111@qq.com`
   - 管理员账号：使用现场已有管理员账号
3. 提前确认 `123@qq.com` 已关联到 `111@qq.com`，用于演示“教师只查看自己的学生”。
4. 建议浏览器开三个标签页，分别用于学生端、教师端、管理员端，减少来回登录的等待时间。

## 二、开场介绍

可以这样说：

“这个项目是面向 GMP 药品生产质量管理课程的实训仿真平台。现在已经完成了学生端学习闭环、教师端学情管理、管理员端基础配置三类核心角色。接下来我会按照学生学习流程、教师查看数据、管理员维护基础数据的顺序演示。”

## 三、学生端演示

### 1. 登录与首次信息完善

演示路径：

1. 使用学生账号 `123@qq.com` 登录。
2. 如果进入首次学习流程，展示“完善信息”页面。
3. 重点说明：
   - 专业不再手动输入，改成选择框，避免填写不统一。
   - 任课老师也改成选择框，学生在前测前选择自己的老师。
   - 选择老师后，教师端就能按老师关联关系查看学生。

讲解口径：

“学生首次进入平台时，需要补全学校、专业、班级和任课老师信息。这里的专业和老师都做成了下拉选择，后续统计和教师端筛选会更稳定。”

### 2. 前测流程

演示路径：

1. 选择学历层次和专业方向。
2. 进入能力前测。
3. 简单答几道题，展示题目进度、选项反馈、上一题/下一题。
4. 提交答卷，展示前测结果。

讲解重点：

- 前测用于判断学生薄弱项。
- 题目会结合学历层次和专业方向。
- 提交后会形成个性化学习方案的基础数据。

### 3. 个性化学习方案

演示路径：

1. 前测提交后进入个性化学习模块。
2. 展示方案中的重点项目、优先级、薄弱项原因和建议。
3. 说明方案包含三类学习安排：
   - 每日练习
   - 课程学习
   - 实训仿真
4. 展示学习计划可下载或后续查看。

讲解口径：

“个性化学习不是简单给一个总分，而是根据前测错题、项目覆盖情况和 AI 建议，整理成每日练习、课程学习、实训仿真的组合计划。”

### 4. 学生首页与学习模块

演示路径：

1. 进入学生首页或主学习页。
2. 展示顶部导航和左侧功能入口。
3. 点开几个核心模块：
   - 每日练习
   - 课程学习
   - 实训仿真
   - 排行榜
   - 背包、技能树、工具
   - 学习报告

讲解重点：

- 页面已做过视觉整理，减少松散布局。
- 点击功能后有轻量反馈，避免用户误以为没有点击。
- 学习报告用于展示通关进度、课时分、学习成果。

### 5. 实训仿真游戏界面

演示路径：

1. 进入实训仿真。
2. 展示顶部导航、右下角 AI 导师面板、好友和学习报告入口。
3. 点击一个任务或项目入口。
4. 展示补给领取、任务反馈、退出提示等交互。

讲解重点：

- 实训部分已经从普通页面转成更偏任务化、游戏化的学习界面。
- 动画做了精简，不做过多干扰，但保留必要反馈。
- 进入和退出项目已经改成更直接的体验，减少中间提示文字打断。

## 四、教师端演示

### 1. 登录教师端

演示路径：

1. 使用教师账号 `111@qq.com` 登录。
2. 进入教师端首页。

讲解口径：

“教师端的重点是看自己学生的数据，而不是看到全平台所有学生。这里已经做了老师和学生的关联。”

### 2. 查看自己的学生

演示路径：

1. 打开学生管理或教师概览模块。
2. 展示学生列表中能看到 `123@qq.com`。
3. 强调当前教师只显示自己关联的学生。

讲解重点：

- 一个老师可以对应多个学生。
- 后续可以继续扩展为一个老师管理多个班级。
- 学生在前测信息完善时选择老师，教师端自动按关系展示。

### 3. 学情统计与项目覆盖

演示路径：

1. 展示教师端统计卡片。
2. 展示前测完成、学习方案、项目任务覆盖、题库等模块。
3. 打开项目任务或知识点覆盖表格。

讲解重点：

- 教师可以查看学生前测情况、学习计划、错题和项目覆盖。
- 表格已优化为尽量一行展示，减少重复换行。
- 教师端标注要求删除的 AI 助手区域已经清理，但相关模块没有误删。

## 五、管理员端演示

### 1. 登录管理员端

演示路径：

1. 使用管理员账号进入管理员端。
2. 先展示管理员首页和个人中心/头像入口。

讲解重点：

- 管理员端负责系统基础数据和资源维护。
- 页面已做过视觉整理，顶部导航、右侧设置和消息入口更加清晰。

### 2. 学校、班级、教师、学生管理

演示路径：

1. 打开学校或班级管理。
2. 展示班级可以配置教师。
3. 展示用户管理中学生、教师、管理员角色。

讲解重点：

- 当前已支持教师与学生的直接关联。
- 班级侧已有教师字段，后续可以继续强化“一个老师管理多个班级”的批量维护能力。

### 3. 项目任务、知识技能、题库资源

演示路径：

1. 打开项目任务覆盖。
2. 展示知识点、技能点、GMP 条款、题库数量。
3. 说明这些数据会支撑前测、练习、课程学习和实训仿真。

讲解口径：

“管理员端更多是平台的底座，维护课程资源、题库、知识技能点和项目任务。学生端看到的学习路径、教师端看到的覆盖情况，都是基于这些基础数据组织出来的。”

## 六、收尾总结

可以这样总结：

“目前系统已经具备了从学生前测、个性化方案、课程练习、实训仿真，到教师查看学情、管理员维护基础数据的主流程。后续重点会继续完善班级批量管理、实训任务完成度记录、学习报告导出和更多真实课程资源接入。”

## 七、演示时注意事项

1. 不要一开始就讲技术细节，先讲业务闭环。
2. 学生端演示要慢一点，重点突出前测和个性化方案。
3. 教师端一定要强调“只看自己的学生”，这是新补的重要逻辑。
4. 管理员端不需要每个表格都点完，展示资源底座即可。
5. 如果某个功能还在完善中，可以说“这里已经完成页面和主流程，后续会继续补充真实数据和批量操作”。
